## Task 4: Add CI build matrix workflow

**Files:**
- Create: `.github/workflows/build.yml`

**Context:** The workflow builds the example app on 6 platform/target combinations across 3 OS runners. `fail-fast: false` so we see every failing job in one run. No analyze/test/dry-run here — those go in `ci.yml` in 0.3.0.

- [ ] **Step 4.1: Verify .github/workflows/ directory state**

Run: `ls -la .github/workflows/ 2>/dev/null || echo "directory does not exist"`
Expected: either lists existing workflows or shows the directory doesn't exist. Either is fine — the next step creates it if needed.

- [ ] **Step 4.2: Create .github/workflows/build.yml**

Create `.github/workflows/build.yml` with this exact content:

```yaml
name: Platform Build Matrix

on:
  push:
    branches: [main]
  pull_request:

jobs:
  build:
    name: Build ${{ matrix.target }} on ${{ matrix.os }}
    runs-on: ${{ matrix.os }}
    strategy:
      fail-fast: false
      matrix:
        include:
          - os: ubuntu-latest
            target: web
            command: flutter build web
          - os: ubuntu-latest
            target: linux
            command: flutter build linux
            setup: sudo apt-get update && sudo apt-get install -y ninja-build libgtk-3-dev
          - os: ubuntu-latest
            target: apk
            command: flutter build apk --debug
          - os: macos-latest
            target: ios
            command: flutter build ios --no-codesign
          - os: macos-latest
            target: macos
            command: flutter build macos
          - os: windows-latest
            target: windows
            command: flutter build windows
    steps:
      - uses: actions/checkout@v4
      - uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.32.7'
          channel: 'stable'
      - name: Platform setup
        if: matrix.setup
        run: ${{ matrix.setup }}
      - name: Pub get (root package)
        run: flutter pub get
      - name: Pub get (example)
        run: flutter pub get
        working-directory: example
      - name: Build
        run: ${{ matrix.command }}
        working-directory: example
```

Key design decisions:
- `fail-fast: false`: see every failing platform in one run
- `flutter build apk --debug`: avoids needing a release signing key
- `flutter build ios --no-codesign`: avoids needing an Apple Developer account
- Linux setup installs GTK3 + ninja (required by `flutter build linux` on Ubuntu)
- Both root and example `pub get`: example imports the root package via `path: ../`
