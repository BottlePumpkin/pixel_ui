# v0.1.1 — Screenshots & pub.dev Polish (Design Spec)

Status: Approved (2026-04-22)
Scope: `pixel_ui` package at repo root.

## 1. Goal & Deliverables

- Ship **5 screenshots** for the pub.dev listing at `doc/screenshots/01_hero.png` ~ `05_texture.png`.
- Each screenshot is generated from a **Flutter widget test (golden)** so regressions are caught in CI / local test runs.
- Register the screenshots in `pubspec.yaml` under `screenshots:` so the pub.dev listing displays them.
- Bump package version to **0.1.1** and update `CHANGELOG.md`.
- Out of scope (explicitly): CI pipeline setup, actual `flutter pub publish`, README screenshot embeds, dark-mode variants, square thumbnails.

**Rationale**
- Scenes live under `test/screenshots/scenes/` so each scene is a single-responsibility `StatelessWidget` and `screenshots_test.dart` stays short.
- Two copies of the PNGs (one under `test/screenshots/goldens/`, one under `doc/screenshots/`) are intentional: tests verify regressions against the `goldens/` copy; pub.dev consumes the `doc/screenshots/` copy. `tool/update_screenshots.sh` keeps them in sync, separating verification from release-artifact handling.
- `doc/screenshots/` is a Dart/Flutter convention and pub.dev reads paths relative to the package root.

## 3. Scene Design (5 screenshots)

All scenes share a common frame:

- Size: **1280 × 720 logical pixels** (`SizedBox`).
- Background: **`#F5F1E8`** (matches `example/lib/main.dart` `scaffoldBackgroundColor`).
- Padding: `EdgeInsets.all(48)`.
- Title label: top of each scene, `PixelText.mulmaru(fontSize: 32)`, label text per scene. **Exception:** `hero_scene` omits the title label because its content already includes a large "PIXEL UI" logo — an extra title would be visually redundant. `_ScreenshotFrame` takes a nullable `title` parameter for this.
- Implementation: `test/screenshots/scenes/_frame.dart` as a `_ScreenshotFrame` wrapper reused by all 5 scenes.

Per-scene content:

| # | File | Contents |
|---|---|---|
| 01 | `hero_scene.dart` | "PIXEL UI" large logo (static version of `_HeroComposition` from example) + 3 small primitives side-by-side (button / box / textured box). Dense, thumbnail-friendly. |
| 02 | `corners_scene.dart` | 7 corners labelled: `sharp`, `xs`, `sm`, `md`, `lg`, `xl`, plus an asymmetric `only(tl:[3,2,1], tr:[3,2,1])` tab. `Wrap` layout. |
| 03 | `shadows_scene.dart` | `sm` / `md` / `lg` shadows on identical yellow boxes, spaced so offset deltas are obvious. |
| 04 | `buttons_scene.dart` | Three `PixelButton`s: normal, **statically-rendered pressed** (passing `pressedStyle` into `PixelBox.style` directly — no tap simulation), and disabled (`onPressed: null`). Labels: `TAP ME` / `PRESSED` / `DISABLED`. |
| 05 | `texture_scene.dart` | Plain vs textured side-by-side with identical corners/fill; the only difference is the texture overlay. |

**Key decisions**
- Pressed button uses static rendering (no `startGesture`/timer) → deterministic goldens, no animation timing flakiness.
- Scenes do not reuse the example app's widget tree. The example is wrapped in `Scaffold`/`AppBar`/`ListView` which is noise for pub.dev thumbnails.
- Color constants are declared `const` at the top of each scene file. Values are **manually synced** with the example app's palette but each scene owns its own declarations (the pub.dev artifact is the source of truth).

## 4. Golden Test Mechanism

`test/screenshots/screenshots_test.dart` iterates over the 5 scenes and runs one `testWidgets` per scene.

Setup per test:
- `tester.view.physicalSize = const Size(2560, 1440);`
- `tester.view.devicePixelRatio = 2.0;` → 1280×720 logical @ DPR 2.0.
- `addTearDown(tester.view.resetPhysicalSize);` + `resetDevicePixelRatio`.
- Wrap scene in `Directionality(textDirection: TextDirection.ltr, child: MediaQuery(data: MediaQueryData(size: Size(1280, 720)), child: scene))`.
- `await tester.pumpAndSettle();`
- `await expectLater(find.byType(_ScreenshotFrame), matchesGoldenFile('goldens/$name.png'));`

**Font loading**
- `setUpAll` loads the bundled Mulmaru font manually (no external dependency on `golden_toolkit`). Because the tests run **inside** the `pixel_ui` package, the asset path is the package-local path (no `packages/<name>/` prefix — that prefix is for consuming apps):
  ```dart
  final loader = FontLoader('Mulmaru')
    ..addFont(rootBundle.load('assets/fonts/Mulmaru.ttf'));
  await loader.load();
  ```
- A single sanity test asserts that a `Text` rendered with `PixelText.mulmaru()` has a non-Ahem width (guards against silent font-load failures).

**Modes**
- `flutter test` (no flag) — compares against committed goldens. Fails on diff.
- `flutter test --update-goldens test/screenshots/` — regenerates `test/screenshots/goldens/*.png`.

**Platform policy**
- Goldens are generated on **macOS**. Subpixel/font-hinting rendering differs across OSes.
- Future CI should pin `runs-on: macos-latest`.
- README note: "Goldens in `test/screenshots/goldens/` are generated on macOS."

## 5. Release Artifact Script

`tool/update_screenshots.sh` — single-responsibility: regenerate goldens, verify, copy to `doc/screenshots/`.

```bash
#!/usr/bin/env bash
set -euo pipefail

cd "$(dirname "$0")/.."

flutter test --update-goldens test/screenshots/screenshots_test.dart

expected=(01_hero 02_corners 03_shadows 04_buttons 05_texture)
for name in "${expected[@]}"; do
  src="test/screenshots/goldens/${name}.png"
  [[ -f "$src" ]] || { echo "Missing golden: $src" >&2; exit 1; }
done

mkdir -p doc/screenshots
for name in "${expected[@]}"; do
  cp "test/screenshots/goldens/${name}.png" "doc/screenshots/${name}.png"
done

echo "✓ Updated 5 screenshots in doc/screenshots/"
```

- Executable bit: `chmod +x tool/update_screenshots.sh`.
- Uses `cp` (no symlinks) so the pub.dev archive contains real files.
- Both `test/screenshots/goldens/` and `doc/screenshots/` are committed.

## 7. Risks & Mitigations

| Risk | Impact | Mitigation |
|---|---|---|
| Platform-specific golden rendering differences (macOS vs Linux font hinting) | CI on another OS produces diffs | Generate on macOS only for 0.1.1. Document in README. Pin `macos-latest` when CI is added later. |
| Mulmaru font fails to load silently → all screenshots render in Ahem | Artifact is unusable | Manual `FontLoader` in `setUpAll` + sanity test asserting non-Ahem text width. |
| Committed PNGs missing from pub.dev archive | Listing has broken images | Release checklist: `flutter pub publish --dry-run` must list all 5 PNGs. |
| Screenshot file size or dimensions exceed pub.dev limits | Upload failure / slow listing | 2560×1440 is within limits. Expected ~100–400 KB each. Script may print `du -sh` for visibility. |
| Scenes drift from example app visually | User confusion | Scenes own their color constants; values are manually synced with the example palette. The scene file is the source of truth for the pub.dev artifact. |
| `screenshots:` YAML schema violation | `pub publish` fails | Run `dart pub global run pana .` locally before publish; its checks cover the screenshots schema and overall pub score. |

## 8. Rollout Order & Definition of Done

Order (dependency-driven):

1. **Scene widgets** — `_frame.dart` + 5 scene files. No goldens yet.
2. **Golden harness** — `screenshots_test.dart` with `setUpAll` font loader. Initial run fails (no goldens exist).
3. **Initial golden generation** — `flutter test --update-goldens test/screenshots/`. Visually inspect all 5 PNGs.
4. **Release script** — `tool/update_screenshots.sh` + `chmod +x`. Run it; confirm `doc/screenshots/*.png` populated.
5. **Metadata bumps** — `pubspec.yaml` version + `screenshots:` block; `CHANGELOG.md` 0.1.1 entry; remove "screenshots deferred" note.
6. **Local verification** — `flutter test` (green), `dart pub global run pana .` (screenshots valid, score ≥ 0.1.0), `flutter pub publish --dry-run` (5 PNGs in archive list).
7. **Commits** (small and narrative-friendly):
   - `test: add screenshot scenes and golden harness`
   - `feat: generate pub.dev screenshots via goldens`
   - `chore: release v0.1.1`

Definition of Done:

- [ ] `flutter test` passes (all goldens match).
- [ ] `tool/update_screenshots.sh` runs clean; 5 PNGs under `doc/screenshots/`.
- [ ] Each PNG visually correct: Mulmaru font rendered, correct colors, sharp pixel edges.
- [ ] `flutter pub publish --dry-run` archive list contains all 5 `doc/screenshots/*.png`.
- [ ] `pubspec.yaml` `screenshots:` paths match real files 1:1.
- [ ] `CHANGELOG.md` has the 0.1.1 entry; "deferred" note removed from 0.1.0.
- [ ] `pana` score does not regress vs 0.1.0.

## Out of Scope (for 0.1.1)

- GitHub Actions / CI pipeline.
- The actual `flutter pub publish`.
- Embedding screenshots in `README.md`.
- Dark-mode or localized screenshot variants.
- A separate square-thumbnail image.
